DJANGO: Vistas

Una View es un lugar donde ponemos la "lógica" de nuestra aplicación. Pedirá información del modelo que has creado antes y se la pasará a la plantilla.Puedes revisar la sección de Templates del curso. Las vistas son sólo métodos de Python que son un poco más complicados que los que escribimos al resolver ejercicios de Programación Orientada a Objetos. Puedes obtener más info sobre views en la comunidad DJANGO.

Las Vistas se colocan en el archivo views.py. Agregaremos nuestras views al archivo ejemplo/views.py.

ejemplo/views.py

Vale, abre este fichero en el editor y mira lo que hay en él:

ejemplo/views.py

from django.shortcuts import render

# Create your views here.

Recuerda que las líneas que comienzan con # son comentarios - significa que Python no las ejecutará.

Creemos una vista (view) como sugiere el comentario. Añade la siguiente mini-vista por debajo:

ejemplo/views.py

def noticia_list(request):

    return render(request, 'ejemplo/post_list.html', {})

Como puedes ver, hemos creado una función (def) llamada noticia_list que acepta request y return una función render que reproduce (construye) nuestra plantilla ejemplo/noticia_list.html.

Guarda el archivo, ve a http://127.0.0.1:8000/ y mira lo que hemos hecho.
